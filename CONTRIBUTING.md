# Contributing to MIS Commentary Agent

Thank you for your interest in contributing to MIS Commentary Agent! This project aims to help finance professionals leverage AI for better MIS reporting and commentary.

## How You Can Contribute

### 1. Prompt Improvements

The core value of this project lies in the quality of prompts. You can contribute by:

- Improving the system prompt in `prompts/system-prompt.md`
- Adding new prompt templates for specific industries
- Enhancing the review checklist in `prompts/review-checklist.md`
- Adding new input/output examples

### 2. Use Cases

Add real-world use cases in `docs/use-cases.md`:

- Manufacturing MIS commentary
- Services/consulting project profitability
- Trading/retail inventory analysis
- Banking/NBFC portfolio commentary
- Healthcare financial reporting

### 3. Code Enhancements

- Build API integrations with LLM providers
- Create a web front-end for file upload
- Add automated quality scoring
- Build a reconciliation module

### 4. Documentation

- Improve the README
- Add tutorials and guides
- Translate documentation to other languages
- Add troubleshooting guides

## Submission Guidelines

### Pull Request Process

1. **Fork** the repository
2. **Create** a feature branch: `git checkout -b feature/your-feature-name`
3. **Make** your changes
4. **Test** thoroughly (especially prompt changes)
5. **Commit** with a clear message: `Add: [brief description]`
6. **Push** to your fork
7. **Submit** a pull request

### Prompt Changes

When submitting prompt improvements:

- Include a **before/after** comparison
- Provide **sample input** and **sample output**
- Test with at least **2-3 different datasets**
- Note any **edge cases** handled

### Code Changes

When submitting code:

- Follow the existing **code style**
- Add **comments** for complex logic
- Include **tests** where applicable
- Update **documentation** as needed

## Reporting Issues

Use the GitHub Issues tab to report:

- **Bugs**: Unexpected behavior in prompts or code
- **Feature Requests**: New agents, modules, or integrations
- **Documentation**: Missing or unclear documentation

When reporting, please include:

1. **Steps to reproduce**
2. **Expected behavior**
3. **Actual behavior**
4. **Environment details** (AI model used, data format, etc.)

## Community Guidelines

- Be **respectful** and **constructive** in all interactions
- Share **credit** when building on others' contributions
- Keep discussions **on-topic** for finance and AI
- No **promotional** content unless it adds clear value
